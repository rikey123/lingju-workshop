# Schema 设计原因说明

## 1. 为什么使用 YAML

YAML 具有层级清晰、可读性强、便于人工修改的特点。相比 JSON，YAML 更适合创作者阅读；相比纯文本剧本，YAML 更适合程序处理和二次开发。

## 2. 为什么设置 metadata

metadata 用于保存标题、版本、语言、格式等信息。这样可以区分不同版本的剧本，也便于后续导出不同格式。

## 3. 为什么设置 source

题目要求小说转剧本，因此必须保留原小说来源信息。source 字段记录小说标题、章节数量、输入方式等内容。

## 4. 为什么设置 characters

剧本中人物会在多个场景反复出现。统一的人物表可以避免名称混乱，也方便进行人物一致性检查。

## 5. 为什么使用 acts

长篇小说可能包含多个剧情阶段。act 可以承载更高层级的剧本结构，适合影视剧、短剧和系列剧扩展。

## 6. 为什么使用 scenes

剧本的基本组织单位是场景，而不是小说章节。scene 字段可以表达地点、时间、出场人物和戏剧目的，是小说改编剧本的核心结构。

## 7. 为什么使用 beats

场景内部不是单一文本，而是由动作、对白、旁白、转场等多个小单元组成。beat 可以让作者细粒度编辑剧本内容。

## 8. 为什么设置 source_trace

AI 改编可能会重写或推断内容。source_trace 可以告诉作者某段剧本来自哪一章、哪一段，增强可信度和可编辑性。

## 9. 为什么设置 conflict_level

剧本强调冲突。conflict_level 可以帮助作者判断场景节奏，也可以用于自动生成情绪曲线和短剧节奏。

## 10. 为什么设置 camera_suggestion

虽然题目只要求剧本 YAML，但镜头建议能体现影视化改编能力，也有利于后续扩展为分镜和视频生成。
