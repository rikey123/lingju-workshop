# YAML Schema 设计说明

## 1. Schema 设计目标

YAML Schema 的目标是规范 AI 输出结果，使剧本初稿具备：

1. 可读性：作者能够直接阅读；
2. 可编辑性：每个场景、对白、动作都能单独修改；
3. 可追溯性：剧本内容能追溯到小说原文；
4. 可扩展性：后续支持短剧、漫画、分镜、视频生成；
5. 可校验性：系统能自动检查 YAML 是否合法；
6. 工程可用性：前端编辑器和后端服务都能读取。

## 2. 顶层结构

```yaml
script:
  metadata:
  source:
  characters:
  story_graph:
  acts:
  quality:
```

## 3. 顶层字段说明

| 字段 | 作用 |
|---|---|
| metadata | 保存剧本标题、版本、语言、生成时间 |
| source | 保存原小说信息 |
| characters | 统一管理人物 |
| story_graph | 保存剧情图谱摘要 |
| acts | 保存幕、场景和 beat |
| quality | 保存质量评分和修改建议 |

## 4. 核心设计思想

### 4.1 小说按章节，剧本按场景

小说章节不一定等于剧本场景，所以 Schema 使用 `acts -> scenes -> beats` 层级。

### 4.2 人物统一管理

人物在 `characters` 中统一定义，场景中只引用人物 ID，避免同一人物名称不一致。

### 4.3 beat 表示场景内部动作

一个场景中可能包含动作、对白、旁白、转场和镜头。使用 beat 可以统一组织这些内容。

### 4.4 source_trace 支持可追溯

每个场景和关键 beat 可以记录来源章节和段落，方便作者回到原文修改。

### 4.5 camera_suggestion 支持扩展

镜头建议不是题目必须要求，但可以体现剧本向影视化、短剧化扩展的能力。
