# 项目总览

## 1. 项目背景

大量小说作者拥有完整故事文本，但不熟悉剧本创作规范。小说以叙述、心理描写和章节推进为主，而剧本强调场景、人物动作、对白、冲突和镜头呈现。因此，小说影视化改编通常需要较高的专业门槛和较长周期。

ScriptMind 希望利用 AI 帮助作者完成从小说到剧本初稿的自动转换，让作者不需要从零学习剧本格式，也能快速得到一个结构清晰、可编辑、可继续打磨的剧本 YAML。

## 2. 项目目标

项目核心目标：

1. 支持输入 3 个章节以上小说文本；
2. 自动识别人物、地点、事件、冲突和对白；
3. 将小说章节转换为剧本场景；
4. 输出结构化 YAML 剧本；
5. 提供 YAML Schema 设计文档；
6. 说明 Schema 的设计原因；
7. 支持作者继续编辑和优化剧本。

## 3. 项目定位

ScriptMind 不是简单的文本摘要工具，而是一个 AI 辅助改编平台。

它的定位是：

```text
小说文本
  ↓
剧情理解
  ↓
人物建模
  ↓
场景重组
  ↓
对白改写
  ↓
剧本 YAML
```

## 4. 目标用户

| 用户类型 | 需求 |
|---|---|
| 小说作者 | 希望将已有小说快速改编成剧本 |
| 网文作者 | 希望尝试短剧、影视化、漫画化改编 |
| 编剧初学者 | 需要结构化剧本初稿作为学习和修改基础 |
| 短剧团队 | 需要快速从故事文本生成可拍摄脚本 |
| AIGC 创作者 | 需要标准化脚本输入后续视频生成工具 |

## 5. 项目价值

### 对作者

- 降低剧本创作门槛；
- 节省改编时间；
- 保留小说原有剧情和人物关系；
- 生成可继续修改的初稿。

### 对创作团队

- 提高前期开发效率；
- 快速评估故事是否适合影视化；
- 支持多种格式扩展，如短剧、漫画、分镜。

### 对技术项目

- 展示 LLM、RAG、多智能体、结构化生成等能力；
- 具备清晰工程实现路径；
- 适合比赛演示和功能扩展。
