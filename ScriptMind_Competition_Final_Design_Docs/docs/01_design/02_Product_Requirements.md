# 产品需求文档 PRD

## 1. 产品名称

ScriptMind

## 2. 产品一句话介绍

ScriptMind 是一个能够将小说章节自动改编为结构化剧本 YAML 的 AI 辅助创作工具。

## 3. 核心需求

### 3.1 小说导入

用户可以上传或粘贴至少 3 个章节的小说文本。

系统需要识别：

- 小说标题；
- 章节标题；
- 章节正文；
- 段落编号；
- 对话内容；
- 叙事内容。

### 3.2 内容解析

系统需要提取：

- 人物；
- 人物关系；
- 地点；
- 时间；
- 事件；
- 冲突；
- 情绪变化；
- 原文关键段落。

### 3.3 剧本生成

系统需要生成：

- 剧本元信息；
- 人物表；
- 幕结构；
- 场景；
- 动作；
- 对白；
- 旁白；
- 转场；
- 镜头建议；
- 来源追踪。

### 3.4 YAML 导出

输出必须是合法 YAML，并符合项目定义的 Schema。

### 3.5 质量评估

系统生成剧本后，给出质量评分和修改建议。

## 4. 非功能需求

| 类型 | 要求 |
|---|---|
| 可读性 | YAML 字段命名清晰，作者可直接理解 |
| 可编辑性 | 每个场景、对白、动作都可单独修改 |
| 可扩展性 | 后续可扩展短剧、漫画、分镜、视频生成 |
| 可追溯性 | 剧本内容能追溯到小说原章节 |
| 稳定性 | 长文本输入时不丢失人物设定 |
| 合法性 | 输出 YAML 必须能通过 Schema 校验 |

## 5. 用户故事

### 用户故事 1

作为小说作者，我希望上传小说前三章，系统能自动生成剧本初稿，以便我快速判断故事是否适合改编。

### 用户故事 2

作为短剧创作者，我希望系统能自动拆分场景和对白，以便我快速制作短剧脚本。

### 用户故事 3

作为编剧，我希望生成的剧本能保留来源章节信息，以便我回到小说原文进行修改对照。
