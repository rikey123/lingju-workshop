# 测试用例设计

## 测试用例 1：正常输入

输入：3 章小说  
预期：成功生成 YAML 剧本。

检查点：

- 是否识别人物；
- 是否生成场景；
- 是否包含 beats；
- YAML 是否合法。

## 测试用例 2：章节不足

输入：2 章小说  
预期：提示“至少需要输入 3 个章节”。

## 测试用例 3：人物名称重复

输入：同一人物有别称  
预期：系统合并人物或提示人工确认。

## 测试用例 4：长篇输入

输入：10 章小说  
预期：系统按章节分批解析，并通过 Character Memory 保持一致性。

## 测试用例 5：YAML 校验

输入：生成后的 YAML  
预期：Schema Validator 返回通过或指出字段缺失。
