# 提交材料清单

## 一、必须提交内容

根据提交规范，项目至少需要包含：

| 材料 | 是否必须 | 本项目对应文件/位置 |
|---|---:|---|
| 公开 GitHub/Gitee 仓库 | 是 | 仓库根目录 |
| Demo 视频 | 是 | README 中放置视频链接 |
| README 文档 | 是 | README.md |
| 可运行代码 | 是 | 后端 FastAPI 服务 + Gradio 演示界面（`python app.py`） |
| 持续开发记录 | 是 | commit 与 PR 历史 |
| 第三方依赖说明 | 是 | docs/05_project_management/03_Dependency_and_Originality.md |

## 二、建议额外提交内容

| 材料 | 作用 |
|---|---|
| 设计文档包 | 提高作品完整度 |
| YAML Schema 文档 | 对齐题目要求 |
| 示例小说与输出 YAML | 证明功能可复现 |
| Demo 视频脚本 | 提升演示表达质量 |
| 测试用例 | 体现开发质量 |
| PR 模板 | 体现工程规范 |
| 架构图与流程图 | 便于评委快速理解 |

## 三、无效风险提醒

以下情况可能导致作品被认为无效或扣分：

1. 仅最后一天一次性提交所有代码。
2. commit 时间不在比赛开发周期内。
3. PR 描述为空或与实际代码严重不符。
4. 使用第三方库但没有在 README 中列明依赖。
5. 复用过往代码但没有说明来源。
6. Demo 视频无法播放。
7. 主分支代码无法运行。
8. 题目要求是 YAML 输出，但项目只输出普通文本。
